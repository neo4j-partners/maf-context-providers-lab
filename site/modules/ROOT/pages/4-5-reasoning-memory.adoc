= Reasoning Memory
:branch: main

Short-term memory captures what was said. Long-term memory captures what was learned. Reasoning memory captures what the agent did and whether it worked. When the agent encounters a similar task later, it can retrieve past reasoning traces to reuse successful strategies and avoid approaches that failed.

== What a Reasoning Trace Contains

A reasoning trace records the full execution path of an agent task:

* **Task** -- A description of what the agent was trying to accomplish
* **Steps** -- Individual reasoning steps, each with a thought, an action, and an observation
* **Tool Calls** -- Specific tool invocations within each step, including arguments, results, status, and duration
* **Outcome** -- The final result of the task
* **Success** -- Whether the task completed successfully

Each trace gets a vector embedding of the task description, enabling semantic search for similar past tasks.

== How Reasoning Memory Integrates

When `include_reasoning=True` is set on `Neo4jMicrosoftMemory`, the context provider injects relevant reasoning traces in `before_run()`. If the current query is similar to a past task, the agent receives that trace as context, including what approach was taken and whether it succeeded.

The `find_similar_tasks` tool (from `create_memory_tools()`) gives the agent explicit access to reasoning traces when it wants to look up past approaches deliberately.

Reasoning memory records agent execution traces including tasks, steps, tool calls, and outcomes.

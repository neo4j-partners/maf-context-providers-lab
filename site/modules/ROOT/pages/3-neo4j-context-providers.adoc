= Neo4j Knowledge Retrieval

In the previous module, you learned how context providers work in the Microsoft Agent Framework and built one from scratch. Now you will connect context providers to Neo4j to build **self-grounding agents** — agents that automatically receive relevant knowledge on every interaction.

Neo4j provides two context provider packages for MAF, each solving a different problem:

* **link:https://github.com/neo4j-labs/neo4j-maf-provider[agent-framework-neo4j^]** — Retrieves knowledge from Neo4j indexes (vector, fulltext, or hybrid search) and injects it before every LLM call. Use this to ground agents in your existing knowledge graph data.
* **link:https://github.com/neo4j-labs/agent-memory[neo4j-agent-memory^]** — Gives agents persistent memory across conversations. Captures user preferences, entity mentions, and reasoning traces during conversations and stores them in Neo4j for future retrieval.

This module covers `agent-framework-neo4j` for knowledge graph retrieval. The next module covers `neo4j-agent-memory` for persistent agent memory.

In this module, you will:

* Understand the `agent-framework-neo4j` package and how `Neo4jContextProvider` extends `BaseContextProvider`
* Use vector search for semantic similarity over movie plot embeddings
* Add graph-enriched retrieval using custom Cypher traversal queries
* _(Optional)_ Use fulltext search for keyword matching without an embedder
* _(Optional)_ Combine both with hybrid search for the best of both approaches

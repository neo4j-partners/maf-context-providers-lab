= Neo4j Agent Memory Context Provider

The context provider from Module 3 injected pre-existing knowledge graph data into agent conversations. That data was static, loaded into Neo4j before the agent ever ran.

The `neo4j-agent-memory` package takes a different approach. It provides a context provider that captures what the agent learns _during_ conversations, including user preferences, entity mentions, and reasoning traces, then stores them in Neo4j for retrieval in future turns and sessions. Like the knowledge retrieval context provider, it hooks into MAF's agent lifecycle through `before_run()` and `after_run()` methods. Before each LLM call, it retrieves relevant memories and injects them into context. After each call, it extracts new information from the conversation and persists it to the graph.

The package provides three memory types (short-term, long-term, and reasoning), a multi-stage entity extraction pipeline, and optional graph algorithm integration. It integrates with the Microsoft Agent Framework through context providers and callable tools.

In this module, you will:

* Understand the three memory types: short-term, long-term, and reasoning
* Learn how the multi-stage entity extraction pipeline works
* Use `Neo4jMicrosoftMemory` as a context provider for automatic memory injection
* Combine passive memory (context providers) with active memory (memory tools)
* Use reasoning memory to learn from past agent executions
* Optionally integrate graph algorithms for advanced entity analysis

= Fulltext Search with Neo4jContextProvider

Vector search finds movies by semantic similarity: "stories about machines becoming conscious" matches movies about artificial intelligence even if those exact words never appear in the plot. Fulltext search works differently. It matches on the actual words in the query. If the user asks about "Christopher Nolan," fulltext search finds plots that contain those exact terms.

Fulltext search does not require an embedder. It uses Neo4j's built-in fulltext indexing with BM25 ranking, making it simpler to set up and faster to execute.

== When to Use Fulltext Search

Fulltext search is the better choice when:

* The user mentions specific names, titles, or terms that should match literally
* You want keyword precision rather than semantic interpretation
* You do not have or need vector embeddings on your nodes
* You want lower latency (no embedding API call required)

Vector search is better when the query is conceptual ("feel-good movies about unlikely friendships") rather than keyword-based.

== Create a Fulltext Index

The recommendations sandbox includes movie plot text on `Movie` nodes. To use fulltext search, you need a fulltext index. If one does not already exist, create it:

[source,cypher]
----
CREATE FULLTEXT INDEX moviePlotsFulltext IF NOT EXISTS
FOR (m:Movie)
ON EACH [m.plot]
----

This indexes the `plot` property of `Movie` nodes for fulltext search.

== Configure the Provider

Set `index_type` to `"fulltext"` and point to the fulltext index. No embedder is needed:

[source,python]
----
import os
from agent_framework_neo4j import Neo4jContextProvider

provider = Neo4jContextProvider(
    uri=os.environ["NEO4J_URI"],
    username=os.environ["NEO4J_USERNAME"],
    password=os.environ["NEO4J_PASSWORD"],
    index_name="moviePlotsFulltext",
    index_type="fulltext",
    top_k=5,
    context_prompt=(
        "## Movie Knowledge Graph Context\n"
        "Use the following movie information from the knowledge graph "
        "to answer questions:"
    ),
)
----

That is the entire setup. No embedder, no embedding model configuration.

== Stop Word Filtering

By default, the provider filters common stop words from the query before searching. Words like "what," "the," "is," and "about" are removed because they add noise to fulltext queries.

For example, "What movies are about time travel?" becomes "movies time travel" before being sent to the fulltext index.

You can disable this with `filter_stop_words=False` if you need exact query passthrough:

[source,python]
----
provider = Neo4jContextProvider(
    uri=os.environ["NEO4J_URI"],
    username=os.environ["NEO4J_USERNAME"],
    password=os.environ["NEO4J_PASSWORD"],
    index_name="moviePlotsFulltext",
    index_type="fulltext",
    filter_stop_words=False,  # Pass raw query to index
    top_k=5,
)
----

== Run the Agent

[source,python]
----
import asyncio
from agent_framework.openai import OpenAIResponsesClient

client = OpenAIResponsesClient(
    api_key=os.environ["OPENAI_API_KEY"],
    model=os.environ["OPENAI_RESPONSES_MODEL_ID"],
)

async def main():
    async with provider:
        agent = client.as_agent(
            name="movie-fulltext-agent",
            instructions=(
                "You are a movie recommendation assistant. Answer questions "
                "using the movie data provided in your context."
            ),
            context_providers=[provider],
        )

        session = agent.create_session()
        response = await agent.run(
            "Tell me about movies with Keanu Reeves",
            session=session,
        )
        print(response)

asyncio.run(main())
----

The query "Tell me about movies with Keanu Reeves" becomes "movies Keanu Reeves" after stop word filtering. The fulltext index finds plots containing those terms and injects them as context.

== Vector vs Fulltext

Compare the same agent with both search modes:

* **"movies about finding meaning in life"** -- Vector search excels here because the concept is abstract. Fulltext search would look for the literal words "finding," "meaning," and "life."
* **"Christopher Nolan Batman"** -- Fulltext search excels here because the query contains specific names that should match literally.

Neither mode is universally better. The right choice depends on how your users query.

= Agent Memory

The context providers from the previous module injected pre-loaded knowledge graph data into your agent's conversations. That data was static. It existed in Neo4j before the agent ever ran.

Agent memory works differently. It captures what the agent learns during conversations, including user preferences, entity mentions, and reasoning traces, then stores them in Neo4j for retrieval in future turns and sessions.

In this module, you will:

* Understand the three memory types: short-term, long-term, and reasoning
* Use `Neo4jMicrosoftMemory` as a context provider for automatic memory injection
* Combine passive memory (context providers) with active memory (memory tools)

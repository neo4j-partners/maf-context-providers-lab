= Neo4j Knowledge Retrieval Context Provider Overview

The context provider you built from scratch in Module 2 managed its own state and injected instructions based on simple logic. Production knowledge graph retrieval needs more than that, including index-based search, embeddings, configurable result formatting, and support for different search strategies.

The link:https://github.com/neo4j-labs/neo4j-maf-provider[`agent-framework-neo4j`^] package provides `Neo4jContextProvider`, a ready-made context provider that connects to a Neo4j database, searches an index, and injects the results before every LLM call. It extends the same `BaseContextProvider` you used earlier but handles the search, embedding, and formatting internally.

Under the hood, `agent-framework-neo4j` delegates all search operations to the link:https://github.com/neo4j/neo4j-graphrag-python[`neo4j-graphrag`^] library. The `neo4j-graphrag` package provides retriever classes -- `VectorRetriever`, `HybridRetriever`, and others -- that know how to query Neo4j vector and fulltext indexes. `Neo4jContextProvider` selects the right retriever based on your `index_type` configuration, wraps it in the MAF lifecycle, and handles the async execution. You configure the provider; `neo4j-graphrag` runs the search.

== How It Works

`Neo4jContextProvider` implements `before_run()` with the following steps:

1. Filters incoming messages to user and assistant roles
2. Takes the most recent messages (configurable via `message_history_count`)
3. Concatenates the message text into a search query
4. Executes a search against a Neo4j index based on the configured `index_type`
5. Formats the results and injects them as context messages via `context.extend_messages()`

The provider doesn't extract entities or keywords from the query. It passes the full message text to the search index and lets Neo4j handle relevance ranking.

== Three Search Modes

The `index_type` parameter determines which search strategy the provider uses:

[cols="1,2,2,1"]
|===
| Mode | How It Works | Best For | Embedder Required?

| `"vector"`
| Embeds the query, searches a vector index by cosine similarity
| Semantic similarity -- finding conceptually related content
| Yes

| `"fulltext"`
| Tokenizes the query, removes stop words, searches a fulltext index by BM25 ranking
| Keyword matching -- finding content with specific terms
| No

| `"hybrid"`
| Runs both vector and fulltext search, combines the scores
| Best of both worlds -- semantic understanding plus keyword precision
| Yes
|===

Each mode maps to a retriever class from the `neo4j-graphrag` library:

* `"vector"` uses `VectorRetriever` (or `VectorCypherRetriever` when a `retrieval_query` is provided)
* `"fulltext"` uses a fulltext retriever that queries Neo4j's Lucene-based fulltext indexes
* `"hybrid"` uses `HybridRetriever` (or `HybridCypherRetriever` with a `retrieval_query`)

You configure the mode when creating the provider. `Neo4jContextProvider` creates the appropriate retriever on connection and delegates every search call to it.

Each provider instance supports a single `index_type`. If you need multiple search strategies -- for example, vector search on one index and fulltext search on another -- create separate provider instances and pass them all to the agent's `context_providers` list. The agent merges the context from every provider before calling the LLM.

== Configuration Options

All three modes share these core parameters:

* **`uri`**, **`username`**, **`password`** -- Neo4j connection details. Falls back to `NEO4J_URI`, `NEO4J_USERNAME`, `NEO4J_PASSWORD` environment variables if not provided.
* **`index_name`** -- The name of the Neo4j index to query. Required.
* **`index_type`** -- One of `"vector"`, `"fulltext"`, or `"hybrid"`. Defaults to `"vector"`.
* **`top_k`** -- Number of results to retrieve. Defaults to 5.
* **`context_prompt`** -- Text prepended to search results to guide the LLM.
* **`message_history_count`** -- Number of recent messages to include in the search query. Defaults to 10.

Mode-specific parameters:

* **`embedder`** -- Required for `"vector"` and `"hybrid"` modes. An object implementing the `neo4j-graphrag` `Embedder` interface.
* **`fulltext_index_name`** -- Required for `"hybrid"` mode. The name of the fulltext index to combine with vector search.
* **`filter_stop_words`** -- For `"fulltext"` mode. Removes common words (articles, prepositions, question words) before searching. Defaults to `True`.
* **`retrieval_query`** -- Optional Cypher query for graph enrichment after index search. Works with all three modes.

== Async Context Manager

The provider manages its Neo4j connection as an async context manager. Use `async with` to ensure the connection is properly opened and closed:

[source,python]
----
async with provider:
    agent = client.as_agent(
        name="my-agent",
        instructions="...",
        context_providers=[provider],
    )
    session = agent.create_session()
    response = await agent.run("query", session=session)
----

The provider connects to Neo4j on entry, verifies connectivity, initializes the appropriate retriever, and closes the connection on exit.

= Building Self-Grounding Agents with Microsoft Agent Framework and Neo4j

== Course Description

Most agent frameworks treat knowledge retrieval as a tool the agent decides to call. The Microsoft Agent Framework takes a different approach: **context providers** run automatically before every agent invocation, injecting relevant knowledge without the agent needing to ask. The result is a **self-grounding agent** — one that consistently receives relevant knowledge from a graph on every interaction, without relying on traditional retrieval tools or custom orchestration code.

This course teaches you how to build self-grounding agents backed by Neo4j. You will:

. Create agents with tools that query a Neo4j graph database.
. Build context providers that inject graph knowledge before every LLM call.
. Use vector, fulltext, and hybrid search with `Neo4jContextProvider` from the `agent-framework-neo4j` package.
. Add graph-enriched retrieval using custom Cypher traversal queries.
. Give agents persistent memory with entity extraction, reasoning traces, and graph algorithms using `neo4j-agent-memory`.

The Microsoft Agent Framework is the direct successor to both Semantic Kernel and AutoGen, built by the same Microsoft teams. Its context provider architecture is a first-class concept with a defined lifecycle, not a plugin or afterthought.

This course uses models from link:https://openai.com/[OpenAI^], although you can use the model and supplier of your choice.

Before taking this course, you should have:

* A basic understanding of Graph Databases and Neo4j
* Knowledge of Python and basic familiarity with async programming

To complete the practical tasks within this course, you will need an link:https://platform.openai.com[OpenAI API key^].

2 hours

=== What you will learn

* What self-grounding agents are and how context providers enable them
* How the Microsoft Agent Framework differs from other agent frameworks
* The context provider lifecycle (`before_run` / `after_run`) and how it differs from tools
* How to build custom context providers that inject dynamic instructions
* How to use `Neo4jContextProvider` with vector, fulltext, and hybrid search modes
* How to add graph-enriched retrieval using custom Cypher traversal queries
* How to give agents persistent memory with entity extraction, reasoning traces, and graph algorithms

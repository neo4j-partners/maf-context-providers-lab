= Graph Algorithm Integration
:branch: main

The memory graph stores entities, relationships, conversations, and reasoning traces as connected nodes. Graph algorithms can analyze this structure to answer questions that simple lookups can't: Which entities are most central to the user's interests? What is the shortest connection path between two entities? Which entities are similar based on their graph neighborhood?

The `neo4j-agent-memory` package includes optional integration with Neo4j Graph Data Science (GDS) that exposes these capabilities as agent tools.

== What GDS Adds

Without GDS, the agent can search entities by name and embedding similarity. With GDS, the agent gains:

* **PageRank** -- Identifies the most important entities based on how many other entities and conversations connect to them. An entity mentioned in many conversations and linked to many other entities ranks higher.
* **Shortest Path** -- Finds connection paths between entities through the graph. "How is Christopher Nolan connected to Blade Runner?" might reveal a path through shared actors or genres.
* **Node Similarity** -- Finds entities with similar graph neighborhoods. Two directors who share actors, genres, or themes will score as similar even if their names never appear together.

GDS configuration options:

* **`enabled`** -- Master toggle for GDS features. Defaults to `False`.
* **`use_pagerank_for_ranking`** -- When `True`, entity search results are ranked partly by PageRank score. More central entities appear higher.
* **`pagerank_weight`** -- How much PageRank influences ranking (0.0 to 1.0). Higher values favor well-connected entities.
* **`fallback_to_basic`** -- When `True` (default), falls back to basic Cypher queries if GDS is not installed in the Neo4j instance.

* **`find_important_entities`** -- Returns the most central entities by PageRank. The agent can answer "What are the most important topics we have discussed?"

== When GDS Is Not Available

If Neo4j GDS is not installed (for example, on a free-tier sandbox), the integration falls back to basic Cypher queries. Shortest path uses Cypher's built-in `shortestPath()`. PageRank and similarity use simpler degree-based approximations. The tools still work, but with reduced accuracy.

Set `fallback_to_basic=True` (the default) to enable this behavior. Set `warn_on_fallback=True` to log warnings when fallback is used.

= Vector Search with Neo4jContextProvider

In the previous lesson, you built a context provider from scratch. That provider injected instructions based on state it managed itself. In this lesson, you will use the `Neo4jContextProvider` from the `agent-framework-neo4j` package, which does the same thing but with knowledge graph data: it searches a Neo4j index and injects matching results before every LLM call.

== Install the Package

[source,bash]
----
pip install agent-framework-neo4j
----

== How Neo4jContextProvider Works

`Neo4jContextProvider` extends `BaseContextProvider` and implements `before_run()` to:

1. Take the user's recent messages
2. Embed them using an embedder (for vector search)
3. Search a Neo4j vector index for semantically similar content
4. Inject the search results as context messages before the LLM is called

The agent receives relevant movie data automatically on every turn, regardless of what the user asks.

== Create an Embedder

Vector search needs an embedder to convert text into vector representations. The `agent-framework-neo4j` package includes an `OpenAIEmbedder`:

[source,python]
----
import os
from agent_framework_neo4j import OpenAIEmbedder

embedder = OpenAIEmbedder(
    api_key=os.environ["OPENAI_API_KEY"],
    model="text-embedding-ada-002",
)
----

The embedder converts the user's query into a 1536-dimensional vector that can be compared against the `plotEmbedding` vectors stored on Movie nodes in your sandbox.

== Create the Context Provider

Configure the provider with your Neo4j connection details and the `moviePlots` vector index:

[source,python]
----
from agent_framework_neo4j import Neo4jContextProvider

provider = Neo4jContextProvider(
    uri=os.environ["NEO4J_URI"],
    username=os.environ["NEO4J_USERNAME"],
    password=os.environ["NEO4J_PASSWORD"],
    index_name="moviePlots",
    index_type="vector",
    embedder=embedder,
    top_k=5,
    context_prompt=(
        "## Movie Knowledge Graph Context\n"
        "Use the following movie information from the knowledge graph "
        "to answer questions:"
    ),
)
----

Key parameters:

* **`index_name`** -- The name of the Neo4j vector index. The recommendations sandbox has a `moviePlots` index on `Movie.plotEmbedding`.
* **`index_type`** -- Set to `"vector"` for semantic search against plot embeddings.
* **`embedder`** -- Generates query vectors using the same model (text-embedding-ada-002) that created the stored embeddings.
* **`top_k`** -- Number of results to retrieve. Five gives a good balance between context richness and token usage.
* **`context_prompt`** -- Text prepended to search results to guide the LLM on how to use the injected data.

== Run the Agent with the Provider

The provider connects to Neo4j using an async context manager. Pass it to the agent's `context_providers` list:

[source,python]
----
async def main():
    async with provider:
        agent = client.as_agent(
            name="movie-knowledge-agent",
            instructions=(
                "You are a movie recommendation assistant. Answer questions "
                "using the movie data provided in your context."
            ),
            context_providers=[provider],
        )

        session = agent.create_session()
        response = await agent.run(
            "What are some good movies about time travel?",
            session=session,
        )
        print(response)

asyncio.run(main())
----

When the agent receives "What are some good movies about time travel?":

1. The framework calls `provider.before_run()`
2. The provider embeds the query and searches `moviePlots` for movies with similar plot descriptions
3. Matching movie data is injected as context messages
4. The LLM generates a response grounded in actual movies from your database

No tool call happened. The context was injected automatically.

== With vs Without Context

Without the context provider, the agent answers time travel questions from its training data. It might mention well-known movies but has no access to your specific database.

With the provider, the agent's response is grounded in the movies that actually exist in your Neo4j graph. It can reference specific plots, mention less obvious films that match semantically, and give answers consistent with your data.

This is the core value of context providers: consistent, automatic knowledge retrieval on every turn.

The correct answer is: **It searches the Neo4j index and injects matching results as context messages.**

The `Neo4jContextProvider` implements `before_run()` to take the user's recent messages, search the configured Neo4j index (vector, fulltext, or hybrid), and inject the matching results as context messages using `context.extend_messages()`. This happens automatically before every LLM invocation.
